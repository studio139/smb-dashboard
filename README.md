# דשבורד פיננסי — סטודיו מרים בסקון

מנוע דטרמיניסטי שמייצר דשבורד אקסל (עברית, RTL, ₪) מתוך ייצואי Fireberry החודשיים.
הקוד מחשב כל מספר; ה-generator בונה כל פריסה. אותו מראה בכל תקופה — גמיש לנתונים.

## מבנה התיקייה
```
CLAUDE.md / README.md         הוראות
references/                    מקורות-אמת: build-brief.md · spec-dashboard-smb.md · style.md
scripts/                       מנוע פייתון (loader · metrics · validator · targets · viewmodel · generator · preview · svg · run · publish · tests/)
templates/                     meeting-template-quarterly.docx — תבנית הוורד הרבעוני
הוצאות קבועות.xlsx            מקור ההוצאות (קובץ אחד, נשאר בשורש, מתעדכן במקום)
inputs/<YYYY-MM>/              5 הייצואים המצטברים של אותו חודש — גם הארכיון (אין העתקה החוצה)
outputs/<שנה>/<סוג>/<תקופה>/  הדוחות: חודשי / רבעוני / שנתי → .xlsx + .html (ללא PDF)
```

## איך מריצים
פותחים טרמינל ומריצים (Python מותקן ב-`C:\Python314`):
```
C:\Python314\python.exe scripts\run.py 2026-04     ← דוח חודשי (אפריל)
C:\Python314\python.exe scripts\run.py 2026-03     ← חודשי + רבעוני Q1 (חודש סוגר — אוטומטי)
C:\Python314\python.exe scripts\run.py 2026-Q1     ← רק הרבעוני (במפורש, לפי דרישה)
C:\Python314\python.exe scripts\run.py 2026        ← רק השנתי (במפורש)
```
**מה מפיקים בכל חודש:** מריצים פשוט את **החודש**. חודש שסוגר רבעון (מרץ/יוני/ספטמבר) מפיק אוטומטית גם את
**הרבעוני**, ודצמבר מפיק גם את הרבעוני (Q4) וגם את **השנתי** — אין מה לזכור. הפלט מאורגן **שנה → סוג →
תקופה**, זהה מקומית וב-Drive: `outputs/2026/חודשי/2026-04/…` · `outputs/2026/רבעוני/2026-Q1/…`. בכל תיקיית
תקופה: `.xlsx` + תצוגת `.html` (דשבורד BI אינטראקטיבי עם טאבים, זהה מספרית לאקסל) — **xlsx + html בלבד, ללא
PDF**. דוח האימות נכתב לשורש הפרויקט (`verification_report.md`); שער הטסטים הוא
`C:\Python314\python.exe scripts\tests\run_tests.py`.

### זרימה חודשית — והזנת הקבצים
1. בכל חודש מייצאים מ-Fireberry את 5 הקבצים ושמים אותם ב-`inputs/<YYYY-MM>/` (למשל `inputs/2026-05/`),
   **באותם שמות בדיוק**. הייצואים **מצטברים** — מניחים את הקובץ המלא, **לא חותכים תאריכים**; המנוע מסנן בעצמו
   לחודש/לרבעון. טווחים:
   - **לקוחות (לידים):** מתחילת 2025 (או אף קודם — כמה שיש), כ-**lookback** ללידים.
   - **פרויקטים · הכנסות · שתפים · משימות:** מ-1.1.2026.
   - כולם **עד סוף החודש הנוכחי**.
2. `הוצאות קבועות.xlsx` — קובץ אחד בשורש, מתעדכן **במקום** (לא מעתיקים לתיקיית החודש).
3. מריצים את הפקודה לחודש הרצוי (ראה הפקודות למעלה). המנוע מאמת, מחשב, בונה, מפרסם ל-Drive ומגבה ל-GitHub.
4. אם משהו חוסם (קובץ/עמודה חסרים, דאטה ריק) — הריצה נעצרת עם הסבר. חוסרים נקודתיים מופיעים ב"דוח חוסרים"
   בתחתית הדשבורד.

> **לידים מ-2025 (או קודם) — מטופלים נכון:** הם **lookback בלבד**, לא נספרים כלידי תקופת הדיווח (2026).
> היסטוריית לידים רחבה יותר **מעלה את שיעור השיוך (attribution) ועשויה להזיז את שיעור ההמרה המיוצב** (יותר
> קוהורטות בשלות נכנסות לחישוב הכותרת) — התנהגות **תקינה וצפויה, לא שגיאה**.

## עיצוב הכותרת
כותרת הדשבורד נושאת את **לוגו הסטודיו** (ב-HTML על "צ׳יפ" לבן בפינת המותג; באקסל בכותרת גיליון
"סקירה"). כרטיסי ה-KPI ממוקמים עם **מרווח נושם** מתחת לסרגל הטאבים. המספרים ולוגיקת העסק לא
השתנו — רק המראה.

## פרסום וגיבוי אוטומטיים
בסוף כל ריצה, אחרי יצירת הדוח, מתבצעים שני צעדים אוטומטיים. שניהם **"רכים"**: אם משהו נכשל
(אין רשת/הזדהות) מודפסת הודעה והריצה ממשיכה — הדוח לעולם לא נחסם.
- **העלאה ל-Google Drive** — קובצי התקופה (`.xlsx` + `.html`) עולים ל-Drive המשותף, לתיקייה
  `דשבורדים/<שנה>/<סוג>/<תקופה>/` תחת "ניתוחים וישיבות הנהלה". מתחבר דרך GWS עם החשבון `studio@smb-arch.com`.
  אם אין הזדהות: מריצים `gws auth login`. *(אפליקציית ה-OAuth כעת ב-Production — האסימון אינו פג
  עוד כל 7 ימים.)*
- **גיבוי ל-GitHub** — כל הפרויקט (קוד · קלטים · הוצאות · מדריך) מקבל `commit` ו-`push` אוטומטי
  ל-repo הפרטי `studio139/smb-dashboard`. תיקיית `outputs/` אינה ב-git (רגנרבילית וכבר ב-Drive).
  כך כל ריצה מגבה את הפרויקט בלי התערבות.

## איך ממלאים את הוורד הרבעוני (יעדים)
בסוף כל רבעון נוצרת תבנית ריקה ב-`inputs/meetings/<רבעון>.docx` (למשל `2026-Q2.docx`).
ממלאים בה:
- **טבלת היעדים** — מספר לכל מדד (פניות · עסקאות · המרה · מחזור · רווח · תקרת הוצאות).
  שמות המדדים מתחברים אוטומטית לדוח → מוצגים כ"בפועל מול יעד".
- **תובנות/יוזמות** — טקסט חופשי (לא נשלף ממנו מספר).
שומרים את הקובץ במקומו. מהריצה הבאה הדוחות יציגו בלוק "עמידה ביעדים".

## עקרונות שאסור להפר
- **דטרמיניזם** — כל המספרים מהקוד.
- **שתי תמונות הכנסה נפרדות** — פייפליין (תמונה א׳) מול תזרים (תמונה ב׳), לעולם לא מספר אחד.
- **Open schema** — ערך חדש (מקור/סוג/קטגוריה/סטטוס/עובד) נקלט אוטומטית כשורה/עמודה חדשה.
- **עברית RTL ו-₪** בכל מקום.
- **הרצה יזומה** — אין schedule עיוור (התזמון נקבע מאוחר יותר באפליקציה).

פירוט מלא: `CLAUDE.md` ו-`references/`.
